Step 1 · Data Generation

☁️
AWS Data Exports
Billing & Cost Management > Data Exports
AWS automatically generates a detailed cost report on a schedule (daily or monthly) covering every service including EC2, RDS, S3 and EKS. Nobody has to manually download anything. The report is produced and sent automatically.
AWS Native
auto-saves CSV file

Step 2 · File Storage

🧻
Amazon S3 Bucket
Simple Storage Service · cloud file storage
The cost report CSV lands here automatically. Think of S3 as a cloud folder. AWS drops the file in and our pipeline picks it up from here. This is the handoff point between what AWS does automatically and what our pipeline does next.
Storage
new file detected · pipeline starts

Step 3 · Orchestration

🌓️
AWS Kiro
Workflow orchestration · the pipeline manager
Kiro detects when a new cost report file arrives in S3 and kicks off the rest of the pipeline in the right order. It tells Glue to start cleaning the data, waits for it to finish, then triggers the next step. This is where the actual coding lives.
Coding required
triggers data cleaning job

Step 4 · Data Cleaning

🧹
AWS Glue
ETL · Extract, Transform, Load
Glue takes the raw CSV file and cleans it up. It removes blank rows, fixes inconsistent formatting, applies the tagging labels, and reshapes everything into a clean standardized format ready for analysis.
ETL
loads clean data into

Step 5 · Central Data Store

🗄️
Central Database
Amazon Athena (recommended for PoC) · or RDS · or Redshift
One single place where all the clean, processed cost data lives. Instead of scattered spreadsheets across different systems, everything is here organized, up to date, and ready to query for reporting and forecasting.
